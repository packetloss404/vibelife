What Changed In This Audit

  • Fixed the combat attack route to pass the real session token into attackEnemy instead of an account id, closing a backend correctness bug in src/routes/combat.ts.
  • Fixed federated identity issuance so the signed payload and returned token use the same issuedAt/expiresAt values in src/world/federation-service.ts.
  • Removed the handshake fallback that accepted a raw challenge echo, tightening federation trust validation in src/world/federation-service.ts.
  • Locked federation server registration behind an authenticated admin session in src/routes/federation.ts.
  • Fixed Godot voxel contract drift in native-client/godot/scripts/world/voxel_manager.gd by supporting chunkX/chunkZ, decoding JSON RLE chunk payloads, syncing palette data, and sending blockTypeId.
  • Fixed Godot initialization bugs by reusing scene-owned voxel/enemy roots, configuring the voxel manager on login, routing initial object loads through ObjectManager.sync_objects(), and correcting WebSocket reconnect cleanup in native-client/godot/scripts/main.gd and native-client/godot/scripts/world/enemy_renderer.gd.
  • Removed obvious AI-slop integration banners from active route files such as src/routes/mobile.ts, src/routes/vr.ts, and src/routes/federation.ts.
  • Converted verification into a non-mutating flow by changing npm run check to run build, tests, and repository checks rather than silently syncing assets first in package.json.
  • Added regression tests for the federation token fix and combat-token bug in src/__tests__/audit-fixes.test.ts.
  • Archived stale review positioning, cleaned the dev-doc index, and made this report the canonical audit entry point.
  • Applied the follow-up product decision that the browser surface is admin/debug only by converting public/index.html and public/app.js to admin-login-only messaging and behavior.
  • Started a partial auth transport cleanup by adding Authorization: Bearer support in src/middleware/auth.ts and using shared token extraction helpers in key route groups.
  • Pruned speculative Godot feature drops out of the primary runtime path by moving them into native-client/godot/scripts/experimental and documenting that they are archived until properly wired and verified.
  • Expanded route deduplication by applying shared auth handling to src/routes/interactives.ts, src/routes/scripts.ts, src/routes/npcs.ts, and token extraction cleanup in src/routes/creator-tools.ts.

Duplication Hotspots

  • Server auth guards: token extraction plus getSession() checks repeat across src/routes/mobile.ts, src/routes/vr.ts, src/routes/npcs.ts, src/routes/scripts.ts, and many others.
  • Route result mapping: many handlers hand-translate ok/reason or null returns into HTTP 400/401/403/404 responses instead of using a shared adapter.
  • Godot HTTP plumbing: request-node creation, JSON parsing, base URL handling, and token/header setup repeat across multiple UI and world managers.
  • Domain literal drift: role, account-kind, season, and other union-like values are still spread across contracts, persistence, and runtime state.
  • Client overlap: browser and Godot clients still duplicate auth, build/edit, parcel, chat, inventory, and moderation concepts with different maintenance costs.

Discussion Items Before Bigger Refactors

  • Decide whether public/app.js remains a real parallel client or is intentionally reduced to a slim debug/admin companion.
  • Decide whether src/world/store.ts is a temporary facade or the long-term application boundary.
  • Standardize auth transport: query/body token support remains widespread and should eventually give way to a single request-auth pattern.
  • Choose one event and payload naming style; contracts currently mix mostly camelCase structures with pockets like duration_ms.
  • Decide which Godot feature modules are production features, which are experiments, and which should be removed or moved to a sandbox path.

Recommended Next Refactor Wave

  1. Adopt shared auth middleware across the highest-traffic route groups.
  2. Extract a route-response helper for common service result patterns.
  3. Break websocket/event handling out of src/server.ts into dedicated command handlers.
  4. Create a shared Godot API helper to remove repeated HTTPRequest setup and JSON parsing logic.
  5. Split native-client/godot/scripts/main.gd into smaller session, networking, world, and UI coordinators.
  6. Add route-level and persistence-mode tests before larger auth/persistence refactors.

Client Direction Decision

  • Browser: treat it as admin-only and internal debug tooling, not a parallel end-user world client.
  • Godot: still viable if the goal is an immersive native viewer with stronger world presence, better input control, and long-term desktop packaging.
  • Godot caveat: the current Godot codebase is carrying prototype debt, speculative modules, and orchestration overload in native-client/godot/scripts/main.gd.
  • If Godot stops fitting: the likely reason would not be rendering quality, but team velocity. If iteration speed on gameplay/admin UX matters more than a native-client feel, a web-first product stack would move faster.
  • Current recommendation: keep Godot as the primary world client for now, but prune or sandbox speculative modules instead of pretending every generated subsystem is production-ready.
  • Current action taken: the most speculative modules now live in native-client/godot/scripts/experimental instead of the active script surface.
