PacketCraft Developer Manual

Architecture, API reference, and internals for the PacketCraft platform — Paper + Fabric + Fastify.

Architecture

PacketCraft is a 3-tier system built for Minecraft server platforms:

Paper Plugin Java

Runs inside the Paper Minecraft server. Handles in-game commands (/parcel, /friends, /balance, /pay, /market), listens to game events, and communicates with the sidecar via HTTP.

Fabric Client Mod Java

Runs on the player's Minecraft client. Provides keybind-activated GUI screens (V/N/M/J/K) for economy, social, marketplace, achievements, and events. Communicates via plugin messages and direct sidecar API calls.

Fastify Sidecar TypeScript

Standalone HTTP server (Fastify 5 + TypeScript) that owns all business logic, persistence, and real-time state. Exposes 40+ REST endpoints. Dual persistence: in-memory for dev, PostgreSQL for prod.

Data Flow

Player Action (in-game)
    |
    v
Paper Plugin (command handler / event listener)
    |  HTTP request
    v
Fastify Sidecar (business logic + persistence)
    |  response
    v
Paper Plugin (chat feedback / scoreboard update)
    |  plugin message channel
    v
Fabric Mod (GUI screen update)

Quick Start

bash# Clone the repository
git clone https://github.com/packetcraft/packetcraft.git
cd packetcraft

# Install Node.js dependencies (sidecar)
npm install

# Start the Fastify sidecar in development mode
npm run dev
# => Listening on http://localhost:3000

# Build the Paper plugin (requires JDK 21+)
cd paper-plugin && ./gradlew build

# Build the Fabric mod (requires JDK 21+)
cd fabric-mod && ./gradlew build
Dev mode uses in-memory persistence — all data resets on restart. Set DATABASE_URL to enable PostgreSQL persistence for production.

Auth Model

PacketCraft supports three auth modes:

  • Guest — No credentials, ephemeral session. Quick for testing.
  • Registered — Display name + password. Persistent account.
  • Minecraft-linked — UUID-based login from the Paper plugin. Automatic account creation on first join.

All auth endpoints return a token that must be passed to subsequent API calls. Tokens identify the session and map to an accountId.

Rate limits are enforced on auth endpoints: 10 req/min for guest and login, 5 req/min for registration, 20 req/min for MC login.

Persistence Layer

The sidecar uses a dual-persistence model:

In-Memory (Development)

Default mode. All data lives in Maps and arrays. Fast iteration, zero setup. Data resets on restart.

PostgreSQL (Production)

Enabled by setting DATABASE_URL. Full schema auto-migration. Same interface, durable storage.

Both backends implement the same Persistence interface, so business logic is storage-agnostic. The persistence layer handles: accounts, avatars, sessions, parcels, objects, inventory, friends, groups, guilds, marketplace, homes, achievements, pets, photos, events, voice, media, and audit logs.

Plugin Message Protocol

The Paper plugin and Fabric mod communicate via Minecraft's plugin message channel system. The channel name is packetcraft:main.

Messages are JSON-encoded and include a type field. The Fabric mod sends requests (e.g., open economy screen), and the Paper plugin relays data from the sidecar back through the channel.

Key message types:

  • economy_data — Balance and transaction info
  • social_data — Friends list and online status
  • achievement_data — Progress and unlocked achievements
  • event_data — Upcoming events and RSVP status
  • pet_data — Active pet state

Paper Plugin Architecture

The Paper plugin is a standard Paper/Bukkit plugin written in Java. Key components:

Entry Point

VibeLifePlugin.java — Main plugin class. Registers commands, listeners, and initializes the sidecar client.

Commands

  • BalanceCommand — /balance
  • PayCommand — /pay <player> <amount>
  • ParcelCommand — /parcel info|claim|release|list
  • FriendsCommand — /friends list|add|remove|messages
  • MarketCommand — /market

Listeners

  • LoginListener — Handles player join/quit, creates sidecar session via /api/auth/mc-login
  • ChatListener — Persists chat messages to the sidecar
  • ParcelListener — Enforces build permissions by checking /api/parcels/check-build
  • AchievementHook — Increments achievement stats on game events

Bridge

  • SidecarClient — HTTP client for communicating with the Fastify sidecar. All API calls go through this class.
  • PluginMessageHandler — Handles plugin message channel for Fabric mod communication
  • AccountIdCache — Caches Minecraft UUID to accountId mapping
  • VaultProvider — Vault economy integration, delegates to sidecar balance endpoints

Fabric Mod Architecture

The Fabric mod is a client-side mod that adds GUI screens and HUD elements. Key components:

Entry Point

VibeLifeClient.java — Client mod initializer. Registers keybinds and network handlers.

Screens

  • EconomyScreen — Balance, transactions, send Vibes (key: V)
  • SocialScreen — Friends, groups, messages, profiles (key: N)
  • MarketplaceScreen — Browse, buy, sell, bid (key: M)
  • AchievementsScreen — Progress, challenges, leaderboards, titles (key: J)
  • EventsScreen — Upcoming events, RSVP, rewards (key: K)

Network

  • PluginChannelHandler — Receives data from the Paper plugin via plugin messages
  • SidecarApi — Direct HTTP calls to the Fastify sidecar for screen data
  • KeybindManager — Registers and handles V/N/M/J/K keybinds

HUD

  • AchievementToast — Toast notification when an achievement is unlocked

Configuration

The Fastify sidecar is configured via environment variables:

VariableDefaultDescription
PORT3000HTTP server port
HOST0.0.0.0Bind address
DATABASE_URL(none)PostgreSQL connection string. If unset, uses in-memory persistence.
CORS_ORIGINSlocalhost:3000Comma-separated allowed origins
API_KEY(none)Shared secret for Paper bridge endpoints

The Paper plugin is configured via config.yml in the plugin data folder:

  • sidecar-url — URL of the Fastify sidecar (default: http://localhost:3000)
  • api-key — Shared secret matching the sidecar's API_KEY

Testing

The sidecar uses Vitest for testing. Tests run against the in-memory persistence layer.

bash# Run all tests
npm test

# Run tests in watch mode
npm run test:watch

# Run a specific test file
npx vitest run src/__tests__/objects.test.ts

Test files are in src/__tests__/. Each test file tests a specific domain (objects, parcels, economy, etc.) using the full sidecar stack.
